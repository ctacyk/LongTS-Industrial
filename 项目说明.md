# LongTS-Industrial：长时序工业故障推理基准与多模态训练

> 简要说明文档 · 反映当前代码库实际状态

## 1. 项目目标

构建一个评估**视觉语言模型（VLM）对长时间序列工业数据推理能力**的基准（LongTS-Industrial），并在该基准上探索 **SFT + GRPO** 的后训练方法，使模型具备从「设备多通道时序监测图」中进行故障诊断的能力。

核心思路：把长时序数据可视化为图像，让 VLM 通过「看图」理解全局趋势，再分层完成感知、异常定位、根因分析、运维决策等任务。

## 2. 任务分层（L1–L4）

| 层级 | 能力 | 典型问题 |
|------|------|----------|
| L1 | 整体描述 | 描述各传感器通道的趋势、波动与相互关系 |
| L2 | 异常定位 | 精确指出异常的时间区间（精确到步） |
| L3 | 根因分析 | 结合多通道推断故障根源及传播路径 |
| L4 | 运维决策 | 给出分级运维建议与优先级 |

## 3. 覆盖设备域

四类工业设备，均使用 TimeBlender 合成多通道时序：

- `coal_mill` 磨煤机
- `pump` 水泵
- `transformer` 变压器
- `wind_turbine` 风力发电机组

## 4. 整体流程

```
真实数据观察/统计标定
        ↓
TimeBlender 物理建模合成多通道时序（热惯性 + 通道耦合 + 故障注入）
        ↓
可视化为 eval.png + 生成 metadata.json（故障类型/区间/因果链）
        ↓
   ┌────────────────────────┬────────────────────────┐
   ▼                        ▼                        ▼
Benchmark 测试集        SFT 训练数据             GRPO 训练数据
(benchmark_tb)         (LLM 生成 L1-L4 答案)    (parquet + reward)
        ↓                                            ↓
多模型评估 (HES 框架)                        Qwen3-VL-8B 后训练
```

## 5. 关键目录与脚本

### 数据合成
- `time_blender/`：合成引擎（`LambdaEvent` + `Generator`，状态化递归实现热惯性与通道耦合）
- `MultiAgentTS/synth_tb_coal_mill.py`、`simulate_faults.py`：各设备域的 TimeBlender 合成与故障注入
- `MultiAgentTS/benchmark_builder.py`：基准数据打包

### 训练数据生成
- `generate_sft_data.py`：用 `qwen-plus` 根据 metadata 生成 L1–L4 的 SFT 训练数据（含质量检查）
- `Data/training/sft_v2/`：SFT 训练/验证集（jsonl）
- `Data/training/grpo_parquet/`：GRPO 训练/验证集（parquet）

### 模型训练
- `configs/sft_qwen3vl_8b.yaml`：Qwen3-VL-8B LoRA SFT 配置（LLaMA-Factory）
- `configs/grpo_config.yaml`：GRPO 配置（基于 SFT 模型，LoRA rank 64，2×A100）
- `MultiAgentTS/training_data/reward_function/htsr.py`：GRPO 奖励函数
- `run_full_train_eval.sh`、`run_server_benchmark.sh`：服务器端训练/评估脚本

### 评估
- `benchmark_eval.py` / `run_benchmark_eval.py`：基于 HES 框架的基准评估
- `htsr_eval.py`：HES 评估核心实现
- `results/benchmark_eval_v2/`：15 个 VLM 的评估结果（含 `domain_level_table.md`）

### 数据质量审计
- `audit_data_quality.py`、`audit_gt_consistency.py`、`audit_snr_full.py`
- `validate_physics.py`、`validate_hes_triplet.py`、`verify_qa_answers.py`

## 6. 评估指标：HES

```
HES = DA × (0.4 × SS + 0.6 × AQ)
```

- **DA**（Diagnostic Accuracy）：诊断方向是否正确
- **SS**（Semantic Similarity）：与参考答案的语义相似度（`text-embedding-v3`）
- **AQ**（Answer Quality）：由 Judge 模型（`qwen-plus`）打分

按 `L1–L4 × 设备域 × 难度` 多维度统计。当前已评估 15 个开源/闭源 VLM（Qwen3-VL 系列、Qwen2.5-VL、InternVL3、MiniCPM-V、GLM-4.6V 等），完整对比见 `results/benchmark_eval_v2/domain_level_table.md`。

## 7. 快速开始

```bash
# 安装依赖
pip install -r requirements.txt

# 生成 SFT 训练数据（需配置 DashScope API Key）
python generate_sft_data.py

# 在基准上评估某个模型
python run_benchmark_eval.py            # 详见脚本内参数

# 服务器端完整训练 + 评估
bash run_full_train_eval.sh
```

> API Key 通过环境变量 `JUDGE_API_KEY` / `VLM_API_KEY` 配置；评估的 Judge 与 Embedding 走 DashScope。

## 8. 重要约定（开发者须知）

项目对数据合成与基准质量有强制规则，见 `.cursor/rules/`：

- **`data-synthesis-loop.mdc`**：所有合成时序必须使用 TimeBlender（禁止纯 numpy 直接造信号），并强制「单样本可视化验证 → 通过后再批量」的闭环。
- **`benchmark-quality-enforcement.mdc`**：图像核验须逐通道描述，故障注入须有物理依据（热惯性、通道耦合、根因传播），所有统计数字必须实跑得出。
